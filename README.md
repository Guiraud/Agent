### 🛡 Privileged Execution
Securely runs root-level commands via a dedicated macOS Launch Daemon. The user approves the daemon once, then the agent can execute commands autonomously via XPC. The Launch Agent runs as the current user (not privileged) and handles everyday shell tasks; only the Launch Daemon runs as root for admin operations.

**SMAppService — OS-Level Code Signature Enforcement**: The Launch Agent and Launch Daemon are registered via Apple's SMAppService framework (macOS 13+). macOS enforces that helpers must be embedded inside the signed app bundle and share the same Team ID — unsigned or differently-signed helpers are rejected by launchd. User approval is routed through System Settings → Login Items & Extensions. This OS-enforced signature chain eliminates the need for manual XPC audit token validation.
